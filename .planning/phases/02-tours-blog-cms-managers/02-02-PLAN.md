<objective>
Build the full Tours CMS: admin table listing with publish/unpublish/delete, a create form, an edit form, and the WaypointBuilder component for managing itinerary coordinates. This is the primary admin CMS feature.
</objective>

<tasks>

<task id="1" type="auto">
**Create Tours Supabase query helpers**

Create `lib/supabase/queries/tours.ts` with:
- `getTours()` — fetch all tours ordered by created_at DESC, columns: id, title, slug, category, duration_days, price_per_person, is_featured, is_published, created_at
- `getTourById(id: string)` — fetch single tour with all fields
- `createTour(data: Partial<Tour>)` — insert tour, return created row
- `updateTour(id: string, data: Partial<Tour>)` — update tour
- `deleteTour(id: string)` — soft delete (set is_published=false) then hard delete
- `togglePublish(id: string, isPublished: boolean)` — toggle publish state

Use `Tour` type from `types/index.ts`.
</task>

<task id="2" type="auto">
**Build WaypointBuilder component**

Create `components/admin/WaypointBuilder.tsx`:

Props:
```typescript
interface WaypointBuilderProps {
  value: Waypoint[];
  onChange: (waypoints: Waypoint[]) => void;
}
interface Waypoint {
  day: number;
  name: string;
  lat: number;
  lng: number;
  description: string;
}
```

UI:
- List of waypoint rows, each with:
  - Day number (auto-incremented, shown as badge)
  - Location Name (text input)
  - Latitude (number input, 4 decimal places)
  - Longitude (number input, 4 decimal places)
  - Description (textarea, 2 rows)
  - Actions: ↑ Move Up | ↓ Move Down | 🗑 Remove (icon buttons)
- "Add Waypoint" button at bottom — appends new empty waypoint
- Helper text below lat/lng inputs: "Find coordinates: Right-click location on Google Maps → copy lat, lng"
- Link button: "Open Google Maps →" (opens maps.google.com in new tab)
- Styled as a card stack with teal left border accent per row

Acceptance criteria:
- File exists at `components/admin/WaypointBuilder.tsx`  
- Add/Remove/Move Up/Move Down all work correctly
- `onChange` called with updated array on every change
</task>

<task id="3" type="auto">
**Build TourForm component**

Create `components/admin/TourForm.tsx` — a comprehensive form using React Hook Form + Zod:

Schema (all required unless noted):
- `title` (text, required)
- `slug` (text, auto-generated from title, editable)
- `short_description` (textarea, max 300 chars)
- `description` (textarea, max 2000 chars, optional)
- `category` (select: spiritual | trek | hill_station | heritage | beach | wildlife)
- `age_groups` (multi-checkbox: senior | family | youth | school)
- `duration_days` + `duration_nights` (number inputs side-by-side)
- `price_per_person` (number, INR)
- `difficulty` (select: easy | moderate | challenging)
- `max_group_size` (number, optional)
- `destinations` (tag input — comma-separated text converted to array)
- `inclusions` (dynamic list — add/remove text items)
- `exclusions` (dynamic list — add/remove text items)
- `hero_image` (ImageUploader, bucket: `tour-images`, folder: `heroes`)
- `images` (multiple ImageUploaders for gallery, max 8 — show as grid of 4)
- `waypoints` (WaypointBuilder component)
- `is_featured` (toggle)
- `is_published` (toggle — shows "Publish" / "Unpublish")
- SEO section (collapsible): `meta_title`, `meta_description`, `og_image`

Layout: Two-column on desktop. Left: main fields. Right: sidebar with image, publish controls, SEO.
Tabs: "Details" | "Itinerary" | "SEO" (use simple tab UI with Tailwind)

Props:
```typescript
interface TourFormProps {
  initialData?: Partial<Tour>;
  onSubmit: (data: Partial<Tour>) => Promise<void>;
  isSubmitting?: boolean;
}
```

Acceptance criteria:
- File exists at `components/admin/TourForm.tsx`
- Form renders all fields listed above
- Validation works (title required, price must be positive, etc.)
- `onSubmit` called with validated data on save
</task>
