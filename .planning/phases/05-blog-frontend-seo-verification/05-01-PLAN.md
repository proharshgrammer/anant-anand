    Export three async functions, each accepting a `supabase: any` typed parameter (matching the pattern from tours.ts that accommodates both server and browser clients):

    - `getPublishedBlogPosts(supabase, options?: { category?: string; page?: number; limit?: number }): Promise<{ posts: BlogPost[]; total: number }>`
      * Query `supabase.from('blog_posts')` with `.select('*', { count: 'exact' })` filtered to `.eq('status', 'published')`
      * Apply `.order('published_at', { ascending: false })`
      * If `options.category` is provided, chain `.eq('category', options.category)` on both the data query and the count query
      * Use `.range(offset, offset + limit - 1)` for pagination (default page=1, limit=10 per D-02)
      * Return `{ posts: data as unknown as BlogPost[], total: count ?? 0 }`
      * On error: `throw new Error(error.message)`

    - `getBlogPostBySlug(supabase, slug: string): Promise<BlogPost | null>`
      * Query `supabase.from('blog_posts').select('*').eq('slug', slug).eq('status', 'published').single()`
      * On error: return `null` (not throw — matches getTourBySlug pattern)
      * Return `data as unknown as BlogPost`

    - `getPublishedToursByIds(supabase, ids: string[]): Promise<Tour[]>`
      * Query `supabase.from('tours').select('id, title, slug, category, duration_days, duration_nights, price_per_person, hero_image, age_groups').in('id', ids).eq('is_published', true)`
      * On error: `throw new Error(error.message)`
      * Return `data as unknown as Tour[]`
      * Accept empty `ids` array and return `[]` without querying (short-circuit)
